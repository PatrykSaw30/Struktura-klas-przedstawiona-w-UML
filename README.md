1. Warto inicjalizować pola już w momencie tworzenia obiektu, ponieważ jest to bardziej efektywne i czytelniejsze niż późniejsze przypisywanie wartości w ciele konstruktora.

2. Diagram UML precyzyjnie określa poziomy dostępu do elementów klasy oraz pozwala rozpoznać, czy dana klasa ma charakter abstrakcyjny.

3. Polimorfizm umożliwia korzystanie z jednej klasy bazowej w taki sposób, że wywoływane są odpowiednie implementacje metod dla różnych typów obiektów.

4. Słowo kluczowe final powoduje, że klasa nie może być dalej rozszerzana, co zapobiega tworzeniu kolejnych klas pochodnych.
